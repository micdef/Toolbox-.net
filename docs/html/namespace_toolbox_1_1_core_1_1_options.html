enum Toolbox.Core.Options.Base64EncodingTable

Specifies the Base64 encoding table to use.

Different encoding tables are suitable for different use cases. The URL-safe variant is recommended for use in URLs and filenames.

Enumerator
Standard 

Standard Base64 alphabet as defined in RFC 4648. Uses '+' and '/' characters, with '=' padding.

Alphabet: A-Z, a-z, 0-9, +, /

UrlSafe 

URL and filename safe Base64 alphabet as defined in RFC 4648 Section 5. Uses '-' and '_' characters instead of '+' and '/'.

Alphabet: A-Z, a-z, 0-9, -, _ This variant is safe for use in URLs and filenames.

◆ RsaPaddingMode

enum Toolbox.Core.Options.RsaPaddingMode

Specifies the RSA encryption padding mode.

The padding mode affects both security and compatibility. OAEP (Optimal Asymmetric Encryption Padding) is more secure than PKCS#1 v1.5.

Enumerator
Pkcs1 

PKCS#1 v1.5 padding.

Legacy padding mode. Compatible with older systems but vulnerable to certain attacks. Not recommended for new applications.

OaepSha1 

OAEP with SHA-1 hash.

More secure than PKCS#1 v1.5. SHA-1 is considered weak; prefer SHA-256 or SHA-384 for new applications.

OaepSha256 

OAEP with SHA-256 hash.

Recommended padding mode for most applications. Provides good security with reasonable performance.

OaepSha384 

OAEP with SHA-384 hash.

Stronger than SHA-256 with slightly higher overhead.

OaepSha512 

OAEP with SHA-512 hash.

Strongest OAEP variant. Recommended for highly sensitive data.

enum Toolbox.Core.Options.SmtpSecurityMode

Specifies the security mode for SMTP connections.

Choose the appropriate mode based on your SMTP server's requirements. When in doubt, use Auto for automatic negotiation.

Enumerator
Auto 

Automatically determine the best security mode. Will attempt STARTTLS if available, otherwise use implicit SSL/TLS.

None 

No encryption (not recommended for production). Use only for local development or internal networks.

StartTls 

Use STARTTLS to upgrade an unencrypted connection. Standard for port 587.

StartTlsWhenAvailable 

Use STARTTLS if available, otherwise continue unencrypted. Less secure than StartTls but more compatible.

SslOnConnect 

Use implicit SSL/TLS from the start. Standard for port 465.
